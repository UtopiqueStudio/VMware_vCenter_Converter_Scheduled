Building the Samples

The Converter Standalone SDK includes one .NET sample that converts a physical machine to a virtual one. You can build C# (.NET) samples using Microsoft Visual Studio 2005 Express and Microsoft Visual Studio 2005. Visual Studio 2003 is not supported, due to a performance issue. See Knowledge Base article 87402 for details.

If there is no Converter Standalone agent installed on the physical machine you want to convert, the sample automatically attempts to install it.

Prerequisites

    The target virtual machine must not exist on the vCenter Server you are logged in.

Building the Sample Using Visual Studio 2005

You can build C# (.NET) samples using Microsoft Visual Studio 2005.

To build the .NET sample using Visual Studio 2005

  • From the Windows Start Menu, open a Visual Studio .NET console window:
    • Select Start Menu –> All Programs –> Visual Studio .NET –> Visual Studio .NET Tools –> Visual Studio .NET Command Prompt
  • Navigate to the SDK\samples\DotNet sub-directory.
  • In the console window, enter Build2005.cmd to build the sample.

Building the Sample Using Visual Studio 2005 Express

The default installation directory of Visual C# Express is c:\Program Files\Microsoft Visual Studio 8. If you have installed Microsoft Visual C# 2005 Express on a different location, you need to create and configure the VSINSTALLDIR system environment variable.

To configure the VSINSTALLDIR system environment variable

  • Create the VSINSTALLDIR system environment variable.
  • Configure the VSINSTALLDIR environment variable with the locations of the Microsoft Visual Studio tools and .NET Framework 2.0. Microsoft Visual Studio tools and .NET Framework 2.0 are located in the Common7 and SDK sub-directories of the Microsoft Visual C# 2005 Express installation directory. Use quotation marks around directory names that have spaces in them. For example:
    "C:\apps\Microsoft Visual Studio 8".

To build the samples using the Visual Studio 2005 Express

  • From the Windows Start Menu, open a Visual Studio .NET console window:
    • Select Start Menu –> All Programs –> Visual Studio .NET –> Visual Studio .NET Tools –> Visual Studio .NET Command Prompt
  • Navigate to the SDK\samples\DotNet sub-directory.
  • In the console window, enter Build2005.cmd to run the file.

Running the Samples

You can run samples from the \bin or \debug directory of a project or using the Visual Studio .NET console. In the console, you can invoke help about an application by running the application without specifying its parameters.

To run a sample using Visual Studio .NET

  • Open the ConverterSamples.sln file in Visual Studio .NET.
  • From the Project menu, select Properties. The Property Pages dialog opens.
  • In the Property Pages dialog, select Configuration Properties.
  • In the left-hand pane, select Debugging.
  • In the right-hand pane, select Command Line Arguments.
  • Click OK to save your changes.
  • Run the sample in the command prompt.

Sample Applications

SampleDescription, command-line syntax, and example...
SubmitWinP2VJob Converts a Windows physical machine into a virtual machine. The sample takes a properties file as an input. A sample properties file is provided along with the sample (submitwinp2vjob.properties). The properties file contains information about the physical source, vCenter Server, Converter Standalone server details, and the conversion job.

Command line syntax:
ConverterSamples <propertiesfile>

Examples:
ConverterSamples submitwinp2vjob.properties
