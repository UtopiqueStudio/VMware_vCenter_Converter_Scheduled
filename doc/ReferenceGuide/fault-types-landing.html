Fault Types Overview

Faults communicate information about errors that occurred when using the VMware vCenter Converter Standalone API.

Faults store information about changes in the system, such as which managed object the fault pertains to.

Faults might include:

  • Which methods might generate this fault.
  • The parent fault of this fault type extends.
  • Child fault types that extend this fault.
  • Description of purpose and usage.
  • Privileges this fault type requires.
  • Parameters this fault type accepts.
  • Values this fault type returns.
  • Faults this fault type throws.

All fault types are listed in the pane to the left of this page. Click a name to see more information. You can also enter part of a fault type name in the Quick Index to find an entry.
