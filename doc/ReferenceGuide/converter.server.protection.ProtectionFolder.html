ConverterServerProtectionFolderMoveInto

Move a set of ProtectionEntities into this ProtctionFolder instance.

This operation is typically used by clients when implementing a drag-n-drop interface to move a set of objects into this folder.

This operation is transactional only with respect to each individual entity. The set of entities is moved sequentially as specified in the list, and committed one at a time. If the moveInto method fails on an object, the method terminates at that point with an exception, leaving the rest of the entities in their original location.

That said, specific logic checks will be done before any moves occur. If any of these logic checks fail then an appropriate fault will be thrown and the tree structure will not have been altered. These checks include checking for duplicate names of entities of the same type within the folder.

ConverterServerProtectionFolderRebootAndResume

Reboot all the protected sources which require it and resume their protection operations. The sources to be reboot will be determined at the time at which this reboot operation is scheduled to occur. The conditions necessary to initiate this operation on an individual source are given in the preconditions for ConverterServerProtectionJobRebootAndResume. The post- conditions for any source that successfully complete this operation are given in the post conditions for ConverterServerProtectionJobRebootAndResume.

Note: If a time for reboot is specified then a VC scheduled task will be created to trigger this operation at the appropriate time. This task will be set in return returned task's info property as vcTask.

ConverterServerProtectionFolderUpdateProtectionParams

Batch updates the protection params for all jobs in this protection group. This operation is transactional only with respect to each individual protection job. The set of protection jobs is updated sequentially, and committed one at a time. If the updateProtectionParams method fails on an object, the method terminates at that point with an exception, leaving the rest of the entities with their original protection settings state.
